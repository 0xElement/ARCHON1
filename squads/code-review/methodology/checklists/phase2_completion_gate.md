# Phase 2 Completion Gate Checklist

A Phase 2 feature report is complete only if all relevant gates pass.

| Gate | Required Evidence | PASS / FAIL / GAP |
|---|---|---|
| Phase 1 intake | Phase 1 feature map and consolidated files loaded | TBD |
| Row reverse-check | Every Phase 1 row has Phase 2 disposition | TBD |
| File coverage | Every Phase 1 file and discovered file reviewed or gap-listed | TBD |
| Method coverage | Security-relevant methods traced | TBD |
| Interface coverage | Web/REST/GraphQL/workers/services/etc covered | TBD |
| Same-functionality | sibling implementations checked | TBD |
| Pattern catalog | every canonical pattern accounted for | TBD |
| Findings quality | only confirmed/potential/suspicious findings reported | TBD |
| CVSS | CVSS included for every finding/candidate | TBD |
| Gaps | unresolved gaps listed honestly | TBD |

## Completion Language

Use one of:

- `Feature review complete`
- `Feature review complete with findings`
- `Feature review incomplete — gaps remain`

Never mark complete if any required ledger row, file, method, interface, or pattern row is unresolved.
